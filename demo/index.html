说明

要运行此演示（在 Chrome 或 Firefox 中），您应该：

  1. 选择现有文件或输入 URL。
    注意选择托管在服务器上的视频，该服务器允许此页面使用 XHR（即使用 CORS）获取这些视频，或在命令行中使用 --test-type 和 --disable-web-security 开关运行 Chrome。
  2. 根据需要更改某些设置：
    • 下载设置：
      • 选择媒体文件下载的块大小。0 表示在一个 XHR 中获取整个文件。
      • 选择两次块下载之间的超时时间（相关比特率会被计算并显示）
      • 您也可以选择实时下载：下载超时根据缓冲区状态和播放速率计算
    • 分段设置：更改用于创建媒体分段的帧数。小数字意味着浏览器通过 MSE 进行大量处理。大数字意味着初始播放前有额外的延迟。
    • 调试设置：
      • 保存结果：将为每个创建的分段在您的计算机上下载一个文件！
      • 日志级别：错误、警告、信息或调试消息输出到调试控制台。
  3. 然后点击"播放"按钮让文件下载并播放，或者单独控制每个步骤：
    1. 点击"加载媒体信息"按钮。将下载块，直到有足够的数据显示文件信息。
    2. 显示文件信息后，选择要播放的轨道。为每个轨道创建 SourceBuffer 或 TextTrack。
    3. 选择轨道后，使用"初始化 Source Buffer"按钮初始化 Source Buffer。（您也可以点击"添加并初始化所有 Source Buffer"。这将选择所有轨道并尝试为其创建 SourceBuffer。可能不适用于所有文件！）
    4. 然后点击"加载媒体数据并播放"按钮继续下载并开始播放。
  4. 任何时候，您可以：
    • 点击"停止媒体下载"按钮，然后在准备好时再次点击"加载媒体数据并播放"。
    • 点击"重置"按钮选择不同的 URL。
    • 使用时间轴在视频中定位。
    • 更改视频的播放速率。

注意事项

  • Chrome 中的 Media Source API 有一些限制，您在使用此演示时可能会遇到：
    • 它不支持超过 1 个视频和 1 个音频
    • 它不支持在现有的 SourceBuffer 初始化后添加新的 SourceBuffer。但您可以删除一些，即使在播放期间。
